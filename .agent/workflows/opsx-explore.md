# /opsx:explore

1. Read the SKILL.md at `.agent/skills/openspec-explore/SKILL.md` for full instructions
2. If a topic is provided, begin investigating; otherwise ask what the user wants to explore
3. Read relevant files in the codebase to understand the current state
4. Analyze patterns, constraints, and dependencies
5. Compare different approaches with pros/cons
6. Present findings and recommendations
7. When the user has clarity, suggest transitioning to `/opsx:propose <change-name>`

**Important**: No artifacts are created during exploration. This is a low-stakes thinking exercise.
